The Underscore Protocol (_Prtrcl)

_Prtcl started as a tool for collective content management inspired in GIT and became a generic protocol for building interoperable applications where content could be stored and forked on multiple platforms and by different users, including web3 DAOs.

This work involved the development of different tools, some of which are listed below:

Intercreativity

A content-management application built on top of _Prtcl. Inspired in tools like Notion or Roam Research but built for interoperability and supporting forking and merging content from different platforms.

We built two versions of this application: One running purely on web2 technologies, and another where content was governed on Ethreum and stored on IPFS.

js-uprtcl

A Typescript library to build _Prtcl-compatible applications. It was divided in different npm packages and included code for handling _Prtcl objects locally and storing them on multiple _Prtcl remotes

Check this documentation site with some details about the architecture and features of js-uprtcl.

Documentation - Github

js-uprtcl-server

A NodeJS + DGraph backend to store and query linked _Prtcl objects.

It stores linked and nested _Prtcl objects and exposes endpoints to efficiently query them.

Github

eth-uprtcl

An Ethereum-based _Prtcl-remote to govern digital content on top of Ethereum and IPFS.

It also includes a prototype for a naming system based on harberger taxes.

Github

reasonable

An Ethereum-based infrastructure to support the organizations behind _Prtcl development. It includes an onchain enforced exit-to community scheme.

Summary - Github
